## Overview

CardioRisk AI is a production-quality machine learning web application that estimates
a patient's 10-year cardiovascular disease risk by combining:

- **Clinical data** — blood pressure, cholesterol, ECG, exercise angina (UCI Heart Disease)
- **Wearable-inspired data** — resting heart rate, daily steps, sleep duration, HRV, stress
- **Engineered features** — lifestyle score, stress index, cardio fitness index, sleep–HR recovery

The system trains multiple models (Logistic Regression, Random Forest, XGBoost), selects the
best by F1-score, and provides SHAP-based explanations for every prediction.

## Quick Start

### 1. Clone & Install

```bash
git clone https://github.com/your-username/cardiorisk-ai.git
cd cardiorisk-ai
pip install -r requirements.txt
```

### 2. Generate the Dataset

Run the production integration pipeline notebook (`CVD_Production_Pipeline.ipynb`)
to produce `outputs/cvd_integrated_dataset.csv`.

### 3. Train & Export Model

```bash
python src/export_model.py
# Optional: specify dataset path
python src/export_model.py --data outputs/cvd_integrated_dataset.csv
```

This creates all files in `models/`.

### 4. Run the App

```bash
streamlit run app.py
```

Open `http://localhost:8501` in your browser.

## Deployment (Streamlit Cloud)

### Step 1 — Push to GitHub

```bash
git init
git add .
git commit -m "initial commit: CardioRisk AI"
git remote add origin https://github.com/YOUR_USERNAME/cardiorisk-ai.git
git push -u origin main
```

> ⚠️ **Important:** Commit the entire `models/` folder (including `.pkl` files).
> Add a `.gitattributes` if files exceed 50 MB (use Git LFS).

### Step 2 — Deploy on Streamlit Cloud

1. Go to [share.streamlit.io](https://share.streamlit.io)
2. Click **New app**
3. Connect your GitHub account
4. Select repository: `YOUR_USERNAME/cardiorisk-ai`
5. Branch: `main`
6. Main file path: `app.py`
7. Click **Deploy**

Deployment takes ~2 minutes. Your app will be live at:
`https://YOUR_USERNAME-cardiorisk-ai-app-XXXX.streamlit.app`

## Common Errors & Fixes

### `model.pkl not found`
**Cause:** `models/` folder not committed to GitHub or export script not run.  
**Fix:** Run `python src/export_model.py` then commit `models/` folder.

### `Feature mismatch / KeyError`
**Cause:** Training and inference use different feature sets.  
**Fix:** `feature_names.json` defines the exact feature order. The app reads this file
automatically and aligns inputs accordingly. Never edit `feature_names.json` manually.

### `pickle.UnpicklingError` or `AttributeError`
**Cause:** Model trained with different scikit-learn version than deployed.  
**Fix:** Ensure `requirements.txt` pins the same `scikit-learn` version used for training.
Check with `pip show scikit-learn`.

### `ModuleNotFoundError: No module named 'xgboost'`
**Cause:** XGBoost not in requirements.txt or not installed on Cloud.  
**Fix:** Verify `xgboost>=2.0.0` is in `requirements.txt` and commit the file.

### App crashes on input
**Cause:** Invalid numeric input or NaN propagation.  
**Fix:** All inputs have default values via `FEATURE_DEFAULTS`. The `build_input_vector()`
function fills any missing field with the dataset median.

### Slow cold start on Streamlit Cloud
**Cause:** SHAP + XGBoost load takes ~15s on free tier.  
**Fix:** `@st.cache_resource` ensures models load only once per session.
